WoW-WLK335部署(基于Azerothcore)

2026-02-24T05:25:32.000Z2026-02-28T06:39:21.253Z2k8 分钟中国

Azerothcore

Azerothcore是一个基于MaNGOS的开源项目，它的目标是提供一个高度可定制的魔兽世界服务器端。Azerothcore的代码仓库可以在这里找到。

1.依赖准备

依赖项 版本要求
Boost ≥ 1.78
MySQL ≥ 5.7.0
OpenSSL ≥ 1.0.x
CMake ≥ 3.16
MS Visual Studio (Community) ≥ 17 (2022)（桌面版）（非预览）

1.1Git

在 Git 安装过程中 —— 调整 PATH 环境。选择"Git from the command line and also from 3rd-party software"选项，确保 Git 命令可以在命令行中直接使用。

1.2Github 桌面

下载安装程序并按步骤进行设置。

1.3Visual Studio

安装编译器/集成开发环境，安装时必须在工作负荷(Workloads)-> 桌面应用和移动应用(Desktop & Mobile) 下选择 使用C++的桌面开发(Desktop development for C++)。

1.4MySQL(推荐使用5.7版本，不要用8.0)

下载安装程序并按步骤进行安装和设置，确保将MySQL添加到环境变量。

1.5OpenSSL(3.0.x Win64 OpenSSL，非 "精简(light)"版)

安装 OpenSSL 时，在选择复制 OpenSSL DLL 的位置时，请选择 OpenSSL 二进制文件 (/bin) 目录。

1.6CMake(x64版本，不要使用RC(候选发布)版本)

下载安装程序并按步骤进行安装和设置，确保将CMake添加到环境变量。

1.7Boost(预编译的Windows二进制程序(Binary))

在 "系统变量"中新建一个名为 "BOOST_ROOT "的环境变量，并将 Boost 安装目录作为变量值，例如 C:/local/boost_1_90_0。重要的是，指向目录时要使用"/"，而不是"\"。确保没有路径末尾斜线。

2.部署Azerothcore

2.1 克隆Azerothcore仓库

打开 Github 桌面应用，点击 “文件” -> “克隆仓库”。在 “URL” 字段中输入 https://github.com/azerothcore/azerothcore-wotlk.git，选择一个本地目录来存储仓库，然后点击 “克隆”。

或者直接使用命令行克隆仓库：

2.2 使用 CMake 配置和生成 Visual C++ 解决方案

  • 1.打开 CMake 应用，点击 “浏览源代码…（Browse Source…）” → 选择Azerothcore源代码目录
  • 2.点击 “浏览构建…（Browse Build…）” → 选择Azerothcore构建目录
  • 3.点击 “配置(Configure)” 按钮，选择 1.3中下载安装的编译器（如Visual Studio 17 2022）作为生成器，如果使用的是 64 位编译器，请务必选择 Win64 版本。
  • 4.确保勾选了使用默认本地编译器（Use default native compilers）
  • 5.确保 TOOLS_BUILD 设置为 all。这将编译稍后设置中所需的提取器。
  • 6.单击生成（Generate）。将生成 Visual C++ 解决方案文件到2中选择的构建目录

2.3 CMake 配置常见错误及解决方案

MySQL 相关错误

  • CMake 未找到 MySQL：

    • 需要设置 MYSQL_INCLUDE_DIR = C:/XX/MySQL/MySQL Server X.X/include 和 MYSQL_LIBRARY = C:/XX/MySQL/MySQL Server X.X/lib(_XX)/libmysql.lib
    • XX 取决于使用的 MySQL 版本
    • 如果在 CMake 中未看到 MYSQL 字段，请勾选 “高级” 复选框

  • 链接器错误（例如 “error LNK2019: unresolved external symbol mysql_server_init”）：

    • 确保 MYSQL_LIBRARY 设置为与编译模式（x64 与 32 位）相匹配的 libmysql.lib

OpenSSL 相关错误

  • CMake 无法找到 OpenSSL：
    1. 选中高级（Advanced）复选框
    2. 在列表中找到两个 OPENSSL 条目，并指向正确的目录：
      • OPENSSL_ROOT_DIR 是安装路径（默认为 C:/OpenSSL-Win32 或 C:/OpenSSL-Win64）
      • OPENSSL_INCLUDE_DIR 是安装路径中的 “include” 文件夹（默认为 C:/OpenSSL-Win32/include 或 C:/OpenSSL-Win64/include）

Boost 相关错误

  • CMake 无法找到 Boost（例如 “Could NOT find Boost (missing: system filesystem program_options iostreams regex)”）：
    1. 找到你的 Boost 文件夹
    2. 运行 Bootstrap.bat 文件
    3. 运行 b2.exe 文件

Git 相关错误

  • 系统上未找到 Git：
    1. 选中高级（Advanced）复选框
    2. 搜索并找到 GIT_EXECUTABLE
    3. 指定 git.exe 的路径，例如 C:/Program Files/Git/cmd/git.exe
    4. 如果没有 git.exe，则需要安装 git（参阅要求）

2.4 编译源代码

  • 1.在 CMake 中打开项目（Open Project），直接用 Visual Studio 打开 AzerothCore.sln 文件。
  • 2.在 Visual Studio 顶部菜单中单击构建（Build），然后选择配置管理器（Configuration Manager）。
  • 3.顶部菜单中单击构建（Build），然后选择配置管理器（Configuration Manager）。
  • 4.在"帮助"下面的列表菜单中，将活动解决方案配置（Active Solution Configuration）平台设置为 x64(如果在 CMake 配置过程中设置了 32 位编译，请选择 x86)，然后单击关闭（设置自动保存）。
  • 5.在右侧边栏的"解决方案资源管理器"中右键单击 ALL_BUILD，然后选择清理（Clean）。
  • 6.右键单击 ALL_BUILD，然后选择构建（Build）。

如果你的图形用户界面未显示解决方案资源管理器，请单击"构建"菜单并选择清理解决方案（Clean Solution），然后选择构建（Build）。
编译完成后，你会在输出中看到类似下面的信息：

========== Build: 22 succeeded, 0 failed, 0 up-to-date, 1 skipped==========

在构建目录中的 bin\RelWithDebInfo或bin\Debug 文件夹(取决于使用Visual Studio时选择的构建类型是RelWithDebInfo还是Debug)，可以找到新编译的二进制文件。
为了让核心正常运行，还需要以下文件：

有两个/三个 DLL 文件需要手动添加到这个文件夹中，需要从以下安装目录中复制它们：

libmysql.dll → C:\Program Files\MySQL\MySQL Server 8.x\lib\libmysql.dll

注意：需要 libmysql 的准确版本来对应你载的 MySQL。因此，不能从网上下载 DLL，需要从mysql文件夹中提取。

版本 1.1.0 之前的 OpenSSL：

libeay32.dll 和 ssleay32.dll → C:\OpenSSL-Win64\ 或 C:\OpenSSL-Win32\ （取决于你的核心是 64 位还是 32 位）。

OpenSSL 1.1.x 名称已更改：

对于 32 位版本：libssl-1_1.dll 和 libcrypto-1_1.dll → C:\OpenSSL-Win32\bin

对于 64 位版本：libssl-1_1-x64.dll 和 libcrypto-1_1-x64.dll → C:\OpenSSL-Win64\bin

OpenSSL 3.0 及以上版本时：

legacy.dll → C:\OpenSSL-Win(32/64)\bin

对于 32 位版本：libcrypto-3.dll 和 libssl-3.dll → C:\OpenSSL-Win32\bin

对于 64 位版本：libcrypto-3-x64.dll 和 libssl-3-x64.dll → C:\OpenSSL-Win64\bin

至此服务端的核心编译完成，如想魔改服务端核心，应在AzerothCore的源代码目录中进行修改，然后使用CMake重新生成Visual Studio项目文件再重新编译。

3. 服务端配置

3.1 提取 DBC, Maps, VMaps 和 MMaps文件

  1. 复制提取工具到客户端文件夹

    • 浏览到编译生成的服务端目录，即 bin\RelWithDebInfo 或 bin\Debug 文件夹
    • 将以下文件复制到魔兽世界客户端文件夹（wow.exe 所在的位置）：
      • map_extractor.exe
      • mmaps_generator.exe
      • vmap4_extractor.exe
      • vmap4_assembler.exe

  2. 复制提取脚本

    • 在 azerothcore-wotlk 源代码目录中的 apps\extractor 文件夹找到 extractor.bat 批处理脚本
    • 将该脚本放到上述客户端文件夹

  3. 执行提取过程

    • 启动 extractor.bat 脚本，选择提取选项
    • 重要提示：
      • 要使服务器正常工作，需要 dbc、maps 和 vmaps ！
      • 不要试图停止 vmaps 提取过程。当控制台打印出 "按任意键…（Press any key…）"时，该过程就结束了。
      • 它会创建两个新文件夹：buildings 和 vmaps。提取时临时生成的 buildings 文件夹在运行后完全无用，可以安全删除。
      • 在第一个任务完成之前不要运行另一个任务，否则会出错。
      • 如果在完成前停止 vmap4extractor，则需要在再次启动前删除 Buildings 目录。
      • 可选但极力推荐：提取 mmaps。在提取过程中，不要试图停止该进程。
    • 完成后操作：
      • 在 bin\RelWithDebInfo 或 bin\Debug 文件夹中新建一个名为 Data 的文件夹
      • 将 vmaps、maps、dbc、mmaps 文件夹移到 Data 文件夹中
